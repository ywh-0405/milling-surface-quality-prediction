# milling-surface-quality-prediction

## 项目背景

这个项目的目标是解决一个机械加工里很直观的问题：

已知铣削时的切削参数、刀具刃数和加工过程中的三向振动信息，能不能预测加工后的表面质量？

当前版本把“表面质量”拆成两部分：

- `Ra`：表面粗糙度，单位通常是微米 `μm`
- `频域特征`：把加工后的表面起伏情况转换到频域后得到的一组特征，用来描述表面纹理在不同频段上的分布

这个仓库不是只做一个神经网络脚本，而是准备成一个以后可以继续扩展的小课题框架：

- 现在可以用模拟数据完整演示流程
- 以后拿到真实实验数据后，可以直接接进来
- 同时保留轻量版和研究版两种使用方式

## 这个项目想回答什么问题

站在加工实验的角度，这个项目想解决的是：

1. 已知某组切削参数时，表面粗糙度会怎么样？
2. 如果再知道加工过程中机床三个方向的振动特征，能不能把预测做得更准确？
3. 除了预测一个 `Ra` 数值，能不能进一步预测表面起伏在频域里的分布特征？

换句话说，项目的核心逻辑是：

`切削参数 + 振动特征 -> 表面粗糙度 + 表面频域特征`

## 输入与输出

### 输入

当前版本默认使用两类输入：

1. 切削参数
   - `spindle_speed_rpm`：主轴转速
   - `feed_per_tooth_mm`：每齿进给量
   - `axial_depth_mm`：轴向切深
   - `radial_depth_mm`：径向切深
   - `tool_flutes`：刀具刃数

2. 三向振动统计特征
   - `vib_x_rms`、`vib_y_rms`、`vib_z_rms`
   - `vib_x_std`、`vib_y_std`、`vib_z_std`
   - `vib_x_peak`、`vib_y_peak`、`vib_z_peak`
   - `vib_x_pp`、`vib_y_pp`、`vib_z_pp`

另外程序会自动补出几个派生量：

- `spindle_freq_hz`：主轴频率
- `tooth_pass_freq_hz`：齿通过频率
- `feed_rate_mm_min`：进给速度

### 输出

当前版本的正式预测目标有两类：

1. `Ra_um`
   - 表面粗糙度

2. `freq_bin_1 ~ freq_bin_8`
   - 表示加工后表面起伏在频域里的 8 个频带特征
   - 它不是振动信号的频谱，而是“表面形貌转换到频域后的表征”

为了避免把“当前演示能力”和“未来扩展方向”混在一起，这里先明确区分：

### 未来已经预留好的

- 接入真实实验汇总表

- 把当前框架扩展成更完整的研究或课程项目

- 适合后期接入真实实验数据

## 数据格式说明

### 方式一：汇总表模式

如果以后每次实验已经整理成一行数据，那么可以直接使用类似下面这样的表：

- `sample_id`
- `spindle_speed_rpm`
- `feed_per_tooth_mm`
- `axial_depth_mm`
- `radial_depth_mm`
- `tool_flutes`
- 三向振动统计特征
- `Ra_um`
- `freq_bin_1 ~ freq_bin_8`

示例文件见：

- [sample_real_data_template.csv](examples/sample_real_data_template.csv)

### 方式二：原始实验文件模式

如果以后每次实验有：

- 三向振动原始时序
- 表面轮廓文件
- 粗糙度测量结果

那么以后也可以先整理进 `data/raw/`，再由程序统一转换成：

- `data/processed/features.csv`
- `data/processed/targets.csv`

这样模型训练部分就不需要改。

## 快速开始

### 1. 准备环境

至少需要：

- Python 3
- `numpy`
- `pandas`
- `scikit-learn`

如果以后要跑研究版，还需要：

- `torch`

### 2. 准备数据

如果你已经有一张整理好的实验汇总表，可以运行：

```bash
python3 -m src.data_io examples/sample_real_data_template.csv data/processed
```

这一步会自动生成：

- `data/processed/features.csv`
- `data/processed/targets.csv`

### 3. 训练轻量版

```bash
python3 -m src.train_lite --features data/processed/features.csv --targets data/processed/targets.csv --out outputs/lite
```

训练后会在 `outputs/lite/` 里写出基础指标文件。

### 4. 研究版接口

当前研究版的入口先用于搭框架和检查模型结构：

```bash
python3 -m src.train_research --n-features 20
```

以后如果接入真实数据，这个入口会继续扩展成完整的研究训练脚本。

## 结果应该怎么看

如果模型训练完成，重点可以看两类结果：

1. `Ra` 预测效果
   - 看预测值和真实值差多少
   - 常见指标包括 `MAE`、`RMSE`、`R²`

2. 表面频域特征预测效果
   - 看预测的 8 个频带特征是否接近真实值
   - 这有助于说明模型不仅能预测粗糙度数值，还能描述表面纹理分布

## 为什么现在还保留原始脚本

因为完全删掉原始脚本并不方便。

保留它们的意义是：

- 可以直接对照原来的作业版本
- 可以看出这次重构不是推翻重来，而是在原有基础上整理
- 如果以后要回头检查最初的公式和原型，也更容易

## 未来如何接入真实实验数据

这是本项目最重要的预留部分。

以后如果有真实实验数据，推荐按下面的思路接入：

1. 每次实验保留唯一编号 `sample_id`
2. 记录该次实验的切削参数
3. 记录三向振动原始信号或至少保存统计特征
4. 保存对应的 `Ra`
5. 如果有表面轮廓，则再把轮廓转换成频域特征
6. 最后统一整理成 `features.csv` 和 `targets.csv`

这样后面的训练代码和预测代码都不用重写。

## 常见问题

### 1. 现在的数据是真实实验数据吗？

不是。当前主要还是模拟数据和原型数据，用来演示整个流程。

### 2. 那为什么还值得做？

因为现在最重要的是先把数据接口、模型接口、说明文档和后续扩展路径搭清楚。等真实数据来了，不需要从零重写。

### 3. 为什么不只预测 `Ra`？

因为只预测一个粗糙度数值还不够。把表面起伏的频域特征也纳入目标，更容易体现表面纹理层面的信息。

### 4. 为什么项目里会同时有旧脚本和新结构？

因为旧脚本适合对照原始思路，新结构适合以后继续扩展，两者同时保留更实用。
